# Architecture

Documentacion tecnica sobre como esta construido el proyecto. Este directorio contiene decisiones de arquitectura, limites entre modulos, estructura de carpetas y convenciones que deben respetarse al evolucionar la plataforma.

Usar este directorio para:

- Explicar decisiones tecnicas vigentes.
- Definir responsabilidades entre `src/app`, `src/features`, `src/modules`, `src/lib`, `src/payload` y `prisma`.
- Documentar limites de dominio antes de agregar features grandes.
- Revisar impacto tecnico antes de iniciar V3.

No usar este directorio para procedimientos operativos, checklists de deploy o estado de tareas; eso vive en `docs/operations` o `docs/project`.
